# 版本管理指南

## 📋 概述

本指南说明如何为项目创建和管理版本号（Git Tags 和 GitHub Releases）。

## 🏷️ 什么是版本号？

- **Git Tag**：代码仓库中的版本标记点
- **GitHub Release**：GitHub上的正式版本发布，可以包含发布说明、下载链接等

## 🚀 创建版本号

### 方法1：使用批处理脚本（Windows，推荐）

直接运行：
```cmd
创建版本标签.bat
```

脚本会引导你：
1. 输入版本号（如：`v1.0.1`）
2. 输入版本说明
3. 自动创建标签并推送到GitHub

### 方法2：手动创建

#### 创建标签

```bash
# 创建带注释的标签
git tag -a v1.0.1 -m "版本 1.0.1：功能描述"

# 推送到GitHub
git push origin v1.0.1

# 或者推送所有标签
git push origin --tags
```

#### 创建GitHub Release

1. 访问：`https://github.com/chenxiaokaiAAAA/aistudio/releases/new`
2. 选择标签（如：`v1.0.1`）
3. 填写发布标题和说明
4. 点击 "Publish release"

## 📝 版本号规范

### 语义化版本（Semantic Versioning）

格式：`主版本号.次版本号.修订号`

- **主版本号（Major）**：不兼容的API修改
  - 例如：`v1.0.0` → `v2.0.0`
  
- **次版本号（Minor）**：向下兼容的功能性新增
  - 例如：`v1.0.0` → `v1.1.0`
  
- **修订号（Patch）**：向下兼容的问题修正
  - 例如：`v1.0.0` → `v1.0.1`

### 示例版本号

```
v1.0.0  - 初始版本
v1.0.1  - 修复bug
v1.1.0  - 新增功能
v1.1.1  - 修复bug
v2.0.0  - 重大更新（可能不兼容）
```

## 🔍 查看版本

### 查看所有标签

```bash
# 查看所有标签
git tag

# 查看标签详情
git show v1.0.0

# 查看标签列表（带说明）
git tag -l -n1
```

### 在GitHub上查看

访问：`https://github.com/chenxiaokaiAAAA/aistudio/releases`

## 📋 版本发布清单

创建新版本前，建议检查：

- [ ] 代码已测试通过
- [ ] 所有更改已提交并推送
- [ ] 更新了版本号
- [ ] 编写了发布说明
- [ ] 检查了重要文件是否完整

## 🔄 完整工作流程

### 1. 开发完成

```bash
# 提交代码
git add .
git commit -m "功能描述"
git push origin master
```

### 2. 创建版本标签

```bash
# 使用脚本
创建版本标签.bat

# 或手动创建
git tag -a v1.0.1 -m "版本 1.0.1：功能描述"
git push origin v1.0.1
```

### 3. 创建GitHub Release（可选）

1. 访问 GitHub Releases 页面
2. 点击 "Draft a new release"
3. 选择标签
4. 填写发布说明
5. 发布

## 💡 最佳实践

1. **定期创建版本**：重要功能完成后创建版本
2. **清晰的版本说明**：说明本次更新的主要内容
3. **遵循版本规范**：使用语义化版本号
4. **标记重要里程碑**：重大更新时创建版本

## 📚 相关命令

```bash
# 删除本地标签
git tag -d v1.0.0

# 删除远程标签
git push origin --delete v1.0.0

# 切换到某个版本
git checkout v1.0.0

# 基于某个版本创建分支
git checkout -b hotfix v1.0.0
```
