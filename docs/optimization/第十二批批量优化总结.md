## 🎯 优化内容

### 1. 订单详情接口 - ShopOrder查询优化 ✅

**文件**: `app/routes/admin_orders_detail_api.py` - `admin_order_detail()`

**优化前**:
- 在循环中为每个订单查询ShopOrder（通过original_order_id和original_order_number）（N+1查询）
- 查询次数：1 + N*2 次（N为订单数）

**优化后**:
- 批量查询所有订单的ShopOrder（通过original_order_id和original_order_number）
- 查询次数：从 1 + N*2 次减少到 3 次

**优化效果**:
- 查询次数减少：从 O(N*2) 降到 O(1)
- 性能提升：预计 85-90%（当订单数量>5时）

---

### 2. 图片清理接口优化 ✅

**文件**: `app/routes/admin_image_cleanup.py` - `cleanup_old_images()`

**优化前**:
- 在循环中为每个订单查询OrderImage（N+1查询）
- 查询次数：1 + N 次（N为订单数）

**优化后**:
- 批量查询所有订单的图片（在循环外执行一次）
- 查询次数：从 1 + N 次减少到 2 次

**优化效果**:
- 查询次数减少：从 O(N) 降到 O(1)
- 性能提升：预计 80-90%（当订单数量>10时）
