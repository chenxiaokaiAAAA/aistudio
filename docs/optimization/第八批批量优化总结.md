## 🎯 优化内容

### 1. 商户订单列表接口优化 ✅

**文件**: `app/routes/merchant.py` - `merchant_dashboard()`

**优化前**:
- 使用 `.all()` 查询所有订单
- 使用 `.all()` 查询各状态订单（用于统计）
- 没有分页支持
- 内存占用大

**优化后**:
- 添加数据库层面分页支持
- 使用数据库聚合函数（COUNT、SUM）统计信息（避免查询所有记录）
- 支持筛选参数（状态、来源类型）
- 返回分页信息

**优化效果**:
- 查询次数减少：从 7 次减少到 3 次（1次分页查询 + 2次聚合查询）
- 内存使用：大幅减少（只加载当前页数据）
- 响应时间：预计减少 60-80%
